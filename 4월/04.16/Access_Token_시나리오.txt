Access Token 시나리오

목표: /todo/list를 확인 
선조건:  사용자는 로그인한 상태 혹은 로그인 후에 Access Token과 Refresh Token을 쿠키로 보관중인 상태 

흐름

1. 서버에 '/api/v1/todos/list'를 호출할 때  Access Token을 헤더의 추가해서 전송 

2. 서버에서 데이터를 보내주고 출력 

대안(예외) 
1-A   - Access Token이 없다. 
   - 서버에서는 401에러가 발생하고 에러메시지는 'No access token' 메세지 전송 
   - React에서  로그인 페이지로 이동 

1-B - Access Token은 존재하는데 만료된 값이다. 
   - 서버에서는 401에러가 발생하고 에러메시지는 'Expired token' 메세지 전송 

   - React쪽에서는 가지고 있는 Refresh Token을 이용해서 다시 갱신 요청 (1-B-1)

   - 서버에서 갱신된 새로운 Access Token과 Refresh Token이 도착 

   - 쿠키에 토큰들을 저장 

   - 원래 요청을 재호출( react query를 사용하면 retry)지정해서 가능 
   
   1-B-1 - Refresh Token 역시 만료되었다 
      - 에러 던지고 React는 로그인 화면으로 이동해야 함
